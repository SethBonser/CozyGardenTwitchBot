Setup Guide

Everything you need to get CozyGardenBot running in your Twitch channel.

📋

Prerequisites

  • Node.js 18+ — nodejs.org
  • A Twitch account for the bot — create a separate account (e.g. MyChannelBot), not your streamer account
  • A Twitch OAuth token for the bot account with chat:read and chat:edit scopes
💡
Get a token easily at twitchtokengenerator.com — log in as the bot account, select Chat Token, and copy the value. It starts with oauth:.
📦

Installation

  1. Clone or download the project 
    git clone https://github.com/your-username/cozy-garden-bot.git
cd cozy-garden-bot
  2. Install dependencies 
    npm install
  3. Create your .env file Copy the example and fill in your values: 
    cp .env.example .env
  4. Start the bot 
    npm start
    You should see CozyGardenBot is awake! in your chat.
⚡

Choose a Mode

CozyGardenBot has two ways to let viewers participate. Pick the one that fits your stream:

🏆 Channel Rewards Mode (WIP)

  • Actions (get seed, water, harvest, expand) are Twitch channel point rewards — free for viewers
  • Viewers still earn and spend 🌸 petals on boosts and upgrades via the shop
  • Set USE_CHANNEL_REWARDS=true in .env

🌸 Petals-Only Mode Recommended

  • All actions cost petals — no channel points needed
  • Viewers start with !startgarden to claim starter petals
  • Set USE_CHANNEL_REWARDS=false in .env

OBS Overlay

The bot includes a real-time pixel-art garden overlay you can drop into OBS as a Browser Source.

  1. Make sure the bot is running The overlay server starts automatically with npm start on port 8080 (or your OVERLAY_PORT).
  2. Add a Browser Source in OBS In OBS → Sources → Add → Browser. Set the URL to: 
    http://localhost:8080/
    Set width to match your garden size. The canvas auto-resizes horizontally as slots are added.
  3. Check the preview Open http://localhost:8080/ in your browser to see the overlay with a checkered preview background. OBS renders it transparent automatically.
🌿
The overlay auto-reconnects if the bot restarts, so you don't need to refresh the Browser Source mid-stream.
📊

Viewer Dashboard

The bot includes a web dashboard viewers can open in their own browser to check their stats, browse the shop, and review their harvest history — no chat commands needed.

What viewers see

  • 🌸 Current petal balance & held seed (with rarity badge)
  • 🌿 Live garden view with pixel-art sprites — synced in real time via WebSocket
  • 🛒 Graphical shop, organized by category
  • 📜 Harvest history with bloom sprites — every plant they harvested, with rarity & petals

How it works

  • Viewers type their Twitch username once — it's saved in their browser
  • Shop purchases made in the dashboard are posted to Twitch chat automatically
  • Garden updates appear instantly via the same WebSocket feed as the OBS overlay
  • ⧉ Popout button opens a compact 430×750 popup — username carries over automatically
  • 🔄 Refresh button in the banner re-fetches all data without logging out
  1. Make sure the bot is running The dashboard is served by the same server as the overlay. Start the bot with npm start — no extra configuration needed.
  2. Access the dashboard locally Open this URL in a browser on the same machine as the bot: 
    http://localhost:8080/dashboard
    This is only visible to you (the streamer). To share it publicly with viewers, follow the Cloudflare Tunnel steps below.

🌐 Public Access via Cloudflare Tunnel (Optional)

Cloudflare Tunnel creates a secure public URL that routes through Cloudflare's network directly to your bot — no router configuration, port forwarding, or Cloudflare account required. The bot spawns it automatically at startup when enabled.

  1. Download the cloudflared binary Go to the Cloudflare downloads page and download the binary for your OS:
    • Windows — download cloudflared-windows-amd64.exe
    • macOS — download cloudflared-darwin-amd64.tgz or install via brew install cloudflared
    • Linux — download the .deb / .rpm package or the raw binary for your architecture
    No account, login, or license key needed — it's a free, single-file download.
  2. Add cloudflared to your PATH The bot launches it by name (cloudflared), so the binary needs to be findable in your shell.
    • Windows — rename the downloaded file to cloudflared.exe, move it to a folder already on your PATH (e.g. C:\Windows\System32 or create a new folder like C:\tools and add that folder to PATH via System Properties → Environment Variables), then verify with: 
      cloudflared --version
    • macOS / Linux — move the binary to /usr/local/bin/cloudflared and make it executable: 
      chmod +x /usr/local/bin/cloudflared
cloudflared --version
    If the version number prints, you're ready.
  3. Enable the tunnel in .env Open your .env file and set: 
    DASHBOARD_TUNNEL=true
  4. Start (or restart) the bot 
    npm start
    Within a few seconds you should see a line like this in the console: 
    🌐 Public dashboard: https://xxxx-xxxx.trycloudflare.com/dashboard
    The bot also posts the URL to your Twitch chat automatically: 
    🌸 Garden Dashboard is live! Open https://xxxx.trycloudflare.com/dashboard ...
  5. Share the URL with your viewers The public URL is valid for the entire session but changes every time the bot restarts. Re-share it at the start of each stream. Viewers can also type !dashboard in chat anytime to get the current link posted automatically.
⚠️
If the bot prints cloudflared not found at startup, the binary either isn't installed or isn't on your PATH. Double-check step 2 above — open a new terminal window after editing PATH so the change takes effect, then run cloudflared --version to confirm it's reachable.
📋
Dashboard env variables at a glance:
DASHBOARD_TUNNEL — false (default) keeps the dashboard local-only; true enables Cloudflare Tunnel (requires cloudflared on PATH).
RAIN_COST — petal cost of the 🌧️ Rain Cloud consumable in the shop (default: 200).
TONIC_COST — petal cost of the 🧪 Growth Tonic consumable in the shop (default: 150).

Channel Rewards Setup

If using Channel Rewards mode, create these rewards in your Twitch Creator Dashboard → Channel Points → Manage Rewards:

Reward NameSuggested CostNotes
Get a Seed100–500 ptsGives the redeeming viewer a random seed (60% common / 30% uncommon / 10% rare). They then use !plant to place it.
Rare Seed500–2000 ptsGives a guaranteed rare plant seed. Set the channel point cost higher than Get a Seed.
Water Plant50–200 ptsWaters a plant in the garden. Viewers can include a slot number in the reward message to target a specific slot.
Harvest Plant100–300 ptsHarvests a bloomed plant and awards petals. Payout is shared with everyone active in chat.
Expand Garden1000–5000 ptsAdds one slot to the shared garden (up to the MAX_GARDEN_SLOTS limit in .env).
📋
After creating each reward, have someone redeem it once with the bot running. The reward's ID will be printed to the terminal — copy it into your .env and restart the bot.

⚙️ Key Configuration Options

These .env variables directly affect how Channel Rewards mode behaves. See the full variable reference above for defaults and all options.

USE_CHANNEL_REWARDS
true (default) — actions are channel point rewards. false — petals-only mode, all actions cost petals.
MAX_GARDEN_SLOTS
The maximum number of slots the garden can expand to (default: 10). The community starts at 3 and grows by purchasing expansions.
ACTIVE_VIEWER_WINDOW_MIN
How recently (in minutes) a viewer must have chatted to receive a share of the harvest payout. Default: 30.
IGNORED_USERS
Comma-separated list of bot usernames to exclude from tracking and payouts (e.g. nightbot,streamelements).
OVERLAY_PORT
Port for the OBS overlay web server. Default: 8080. Add http://localhost:8080/ as a Browser Source in OBS.
